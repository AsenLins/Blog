初探Hexo

Hexo是一个可以解析markdown语法的静态博客框架。

安装NodeJs

因为Hexo是基于node,所以需要先安装node和npm,这里建议直接安装nvm去管理node和npm,然后再去安装cnpm,cnpm是使用了淘宝镜像服务的npm,安装了之后，下载包的速度会大大提升,因为访问的是大陆的服务器，npm的服务器在国外。

  • nvm的安装教程
  • cnpm安装教程

安装Hexo-cli,Hexo-cli是hexo的一个官方命令工具。

部署Hexo到Github Page

创建仓库并设置默认分支为master


设置默认分支为master

修改仓库名称

在Github的仓库首页找到setting,这里我们修改为blog,你也可以修改为其他名称。

Github仓库升级为Github Page

在选中的仓库中进入setting

在seting页面找到开通Github Page的配置,点击Upgrade

开通后当前页面会刷新,往下滚动,分支选中master然后点击save

到这里,Github Page 就配置好了

验证Github Page

在Github仓库主页上添加index.html



GitHub Page的域名可以在setting的GitHub Page的设置项中找到

然后输入你的Github Page网址,我这里的地址是:https://asenweb.github.io/blog/index.html ,请根据自身的GitHub Pages地址进行访问。

注意:Github Pages 已经被墙了，如果没有自己的域名的话，只能用梯子访问。

部署到Github Page

在hexo生成的根目录中找到_config.yml添加以下配置:

  • url:就是自己的GitHubPage地址
  • root:就是自己的仓库名称
  • type:部署类型
  • repo:就是你git仓库地址
  • branch:就是仓库地址分支，一般都是master

自定义Github Page域名

  1. 在阿里云购买域名并解析到GitHub Page的域名,解析域名等阿里云和网上都有教程，这里就不复述了。

  2. 在hexo根目录中的_config.yml中修改配置

  • url修改为你购买的域名
  • root修改为/
  1. 在hexo目录的source文件夹添加一个命为CNAME文件夹,并添加你购买的域名

  1. 运行部署命令hexo g -d,重新部署即可

到这,绑定个人域名就成功。

常用Hexo命令

  • hexo new “新的文章名称”
  • hexo new page “新的模板页面”
  • hexo s -debug 本地运行debug模式
  • hexo s 本地运行
  • hexo g -w 本地即时构建
  • hexo g -d 构建并部署到远程服务器
