nvm是什么?

nvm(NodeJs version Manger) 是用于管理NodeJs版本控制器,用于切换NodeJs版本。

nvm安装和配置

  1. 找到nvm-noinstall.zip然后下载
  2. 解压nvm-noinstall.zip
  3. 进入到解压的目录
  4. 找到并运行install.cmd,然后会要你输入路径,直接回车就好。
  5. 新建一个settings.txt文本,文本内容如下所示：

保存即可。

  • root:就是你解压nvm的所在路径,也就是nvm的根目录。
  • path:就是用于存储当前使用NodeJs的版本路径（自己选择存放在哪里）。
  • arch:系统是32位就填32,64就填64。
  • proxy:代理,没有就填none。
  • node_mirror:设置安装node的镜像地址,这里设置为淘宝的镜像地址,加快下载node的速度
  • npm_mirror:设置安装npm的镜像地址,这里设置为淘宝的镜像地址,加快下载npm的速度
  1. 安装好后,打开系统环境变量设置,在系统变量中新建NVM_HOME，把值设置为settings.txt的root值,然后再新建NVM_SYMLINK,把值设置为settings.txt的path的值,再找到path变量,添加 ;%NVM_HOME%;%NVM_SYMLINK%;

上面是win10的环境变量设置,win7的请去找相对应的设置教程。

  1. 打开CMD命令行工具,输入nvm v 查看版本,如果有版本信息，则代表安装成功。

注意的地方:

  • 如果你之前已经安装过nodeJs版本,请先删除之前的NodeJs,再删除在环境变量中的对应的路径。

  • nvm的配置是根据settings.txt文件来执行的,root就是当前nvm的执行路径,所下载的node版本都会在这里面,path就是当前使用的node的路径,每当你使用nvm use x.x.x指令的时候就会把对应的nodeJs版本放入该目录中，所以，如果需要目录变动,只需要修改settings.txt里配置即可。

  • 若你的系统为32位,请在安装和使用的指令中加入以下参数

    nvm install 4.4.4 32

nvm use 4.4.4 32
